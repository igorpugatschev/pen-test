# Занятие 17d. OWASP A06: Vulnerable and Outdated Components — Уязвимые компоненты

## Теория

**A06:2021 – Vulnerable and Outdated Components** — использование устаревших библиотек, фреймворков и ПО с известными уязвимостями (CVE).

### Почему это опасно

1. **Известные CVE** — эксплойты уже доступны в интернете
2. **Автоматизированные атаки** — сканеры массово ищут уязвимые версии
3. **Цепочки атак** — сначала эксплуатируется компонент, затем повышаются привилегии

### Где искать компоненты

| Компонент | Как определить версию |
|-----------|---------------------|
| Веб-сервер | Заголовок `Server` в HTTP-ответе |
| CMS (WordPress, Drupal) | Файл `readme.html`, meta-теги, пути `/wp-admin/` |
| JS-библиотеки | Просмотр исходного кода, пути в `/js/` или `/vendor/` |
| Фреймворк (Laravel, Django) | Заголовки `X-Powered-By`, ошибки |
| Библиотеки (jQuery, Bootstrap) | Пути к файлам, версии в комментариях |

### Пример: Уязвимый jQuery

Версия jQuery 1.8.3 содержит уязвимость XSS (CVE-2011-4969). Если сайт подключает:
```html
<script src="https://site.com/js/jquery-1.8.3.min.js"></script>
```
Атакующий может использовать уязвимость для выполнения XSS.

## Практическое занятие

### Сканирование DVWA на устаревшие компоненты

**Шаг 1: Определение версий через заголовки**
1. Откройте Burp Suite → Proxy
2. Перехватите запрос к http://192.168.0.x
3. Посмотрите заголовки ответа:

Пример вывода:
```
HTTP/1.1 200 OK
Server: Apache/2.4.25 (Debian)
X-Powered-By: PHP/5.6.30
Set-Cookie: PHPSESSID=abc123; path=/
```

Версии:
- Apache 2.4.25 (выпущен в 2016 году!)
- PHP 5.6.30 (EOL — поддержка прекращена)

**Шаг 2: Поиск JS-библиотек**
1. Откройте http://192.168.0.x, нажмите Ctrl+U (View Source)
2. Найдите подключенные скрипты:

```html
<script src="http://192.168.0.x/vulnerabilities/xss_r/source/jquery.js"></script>
<script src="http://192.168.0.x/vulnerabilities/xss_r/source/jquery-1.10.2.min.js"></script>
```

jQuery 1.10.2 — устаревшая версия с известными уязвимостями.

**Шаг 3: Поиск CVE через поиск**
Откройте https://cve.mitre.org или https://nvd.nist.gov

Поиск: "Apache 2.4.25 CVE"
Результат:
```
CVE-2017-3167 - Apache HTTP Server: Authentication bypass
CVE-2017-3169 - Apache HTTP Server: Use-after-free
CVE-2017-7668 - Apache HTTP Server: Buffer overflow
```

**Шаг 4: Использование Wappalyzer (браузерное расширение)**
1. Установите Wappalyzer в браузер
2. Откройте http://192.168.0.x
3. Нажмите на иконку Wappalyzer

Результат:
```
Apache 2.4.25
PHP 5.6.30
MySQL
jQuery 1.10.2
```

## Частые ошибки

1. **Игнорировать зависимости** — использовать старые библиотеки из CDN
2. **Не следить за CVE** — не подписываться на уведомления об уязвимостях
3. **Копировать старый код** — использовать примеры с устаревшими версиями библиотек
4. **Забывать о транзитивных зависимостях** — уязвимость в библиотеке, которую использует ваша библиотека

```bash
# Сканирование уязвимостей в Docker-образах (работает на M2)
docker run --rm -v /var/run/docker.sock:/var/run/docker.sock \
  aquasec/trivy image vulnerables/web-dvwa

# Поиск CVE через список версий
curl -s "https://cve.circl.lu/api/search/apache" | jq '.[:5]'

# Установка Wappalyzer CLI (через npm, работает на M2)
npm install -g wappalyzer-cli
```

## Адаптация под macOS (M2, 8GB)

- Для установки инструментов используйте Homebrew: `brew install <tool>`
- На MacBook Air M2 (8GB) запускайте VM с памятью не более 3-4GB
- Используйте UTM вместо VirtualBox (лучшая поддержка ARM)
- Docker работает нативно на M2: `docker pull <image>`
- Для VPN используйте Tunnelblick (OpenVPN) или официальные клиенты
- Для Python используйте `pip3 install` вместо `pip install`
