# Занятие 14. Маршрутизация: таблицы маршрутов, traceroute, route

## Теория

Маршрутизация — процесс выбора пути для передачи пакетов от источника к получателю через сеть.

### Таблица маршрутизации

Таблица маршрутизации содержит правила, по которым определяется, куда отправлять пакеты.

**Основные поля таблицы маршрутов:**
- **Destination** — сеть назначения
- **Gateway** — шлюз (следующий узел)
- **Genmask** — маска сети
- **Flags** — флаги (U — up, G — gateway, H — host)
- **Iface** — исходящий интерфейс

### Типы маршрутов

**Прямой маршрут (Connected)** — сеть подключена непосредственно к интерфейсу
```
192.168.1.0     0.0.0.0         255.255.255.0   eth0
```

**Маршрут через шлюз (Gateway)** — пакеты отправляются на роутер
```
0.0.0.0         192.168.1.1     0.0.0.0         eth0
```
(0.0.0.0/0 — маршрут по умолчанию, default gateway)

**Хостовый маршрут** — маршрут до конкретного IP
```
10.10.10.5      192.168.1.1     255.255.255.255 eth0
```

### Процесс маршрутизации пакета

1. Определение IP-адреса назначения
2. Проверка локальных адресов (loopback, локальные интерфейсы)
3. Поиск в таблице маршрутов (самый специфичный маршрут — больше префикс)
4. Если маршрут найден — отправка через указанный интерфейс/шлюз
5. Если не найден — отправка через default gateway
6. Если нет default gateway — ошибка "Network unreachable"

### Протоколы маршрутизации

**Внутридоменные (IGP):**
- **RIP** (Routing Information Protocol) — дистанционно-векторный, метрика — количество хопов
- **OSPF** (Open Shortest Path First) — состояние канала, метрика — пропускная способность
- **EIGRP** (Cisco proprietary) — гибридный

**Междоменные (EGP):**
- **BGP** (Border Gateway Protocol) — используется в интернете

### traceroute

traceroute показывает маршрут до цели, определяя промежуточные узлы (роутеры).

**Как работает traceroute:**
1. Отправляет пакеты с TTL=1, получает Time Exceeded от первого роутера
2. TTL=2 — от второго, и так далее
3. Когда TTL достаточно — получает ответ от цели

### Для пентестера

- Определение топологии сети
- Обход фильтрации (если известны альтернативные маршруты)
- Атаки на протоколы маршрутизации (BGP hijacking, RIP poisoning)
- Использование traceroute для reconnaissance

## Практическое занятие

### Задача 1: Просмотр таблицы маршрутизации

1. Просмотрите текущую таблицу маршрутов:
```bash
ip route show
# или классический вариант
route -n
```

2. Более детальный вывод:
```bash
ip route list
```

3. Просмотр кэша маршрутизации (если поддерживается):
```bash
ip route show cache
```

### Задача 2: Добавление и удаление маршрутов

1. Добавьте статический маршрут до тестовой сети:
```bash
sudo ip route add 10.10.10.0/24 via 192.168.1.1
```

2. Проверьте, что маршрут добавился:
```bash
ip route show | grep 10.10.10
```

3. Добавьте маршрут через конкретный интерфейс (без шлюза):
```bash
sudo ip route add 172.16.0.0/16 dev eth0
```

4. Удалите добавленные маршруты:
```bash
sudo ip route del 10.10.10.0/24
sudo ip route del 172.16.0.0/16
```

### Задача 3: Использование traceroute

1. Установите traceroute:
```bash
# macOS
brew install traceroute
# Linux
sudo apt install traceroute
```

2. Выполните traceroute до публичного хоста:
```bash
traceroute 8.8.8.8
```

3. Используйте TCP-трассировку (часто обходит фильтры):
```bash
sudo traceroute -T -p 80 8.8.8.8
```

4. Используйте ICMP (как ping):
```bash
sudo traceroute -I 8.8.8.8
```

### Задача 4: Альтернативы traceroute

1. Используйте mtr (комбинация ping и traceroute):
```bash
# macOS: brew install mtr
# Linux: sudo apt install mtr
brew install mtr
mtr 8.8.8.8
```

2. Используйте tcptraceroute:
```bash
# macOS: brew install tcptraceroute
# Linux: sudo apt install tcptraceroute
brew install tcptraceroute
tcptraceroute 8.8.8.8
```

3. Трассировка через curl (только для HTTP):
```bash
curl --max-time 10 -v http://example.com 2>&1 | grep -i "trying"
```

## Частые ошибки

1. **Путаница метрик маршрутов**: Когда есть несколько маршрутов до одной сети, выбирается маршрут с большей специфичностью (больший префикс, например /24 важнее /16).
2. **Забыли включить IP forwarding**: Для работы как роутер нужно `sysctl net.ipv4.ip_forward=1`. Без этого транзитные пакеты отбрасываются.
3. **route vs ip route**: Команда `route -n` устарела, используйте `ip route show` для просмотра таблицы маршрутизации.
4. **traceroute и файрволы**: Traceroute по умолчанию использует UDP. Если не работает, попробуйте `-T` (TCP) или `-I` (ICMP).

## Вопросы на понимание

1. В каком порядке просматривается таблица маршрутизации?
    <details><summary>Ответ</summary>Сначала проверяются самые специфичные маршруты (больший префикс, например /32), затем менее специфичные, в конце — default gateway (0.0.0.0/0)</details>
2. Зачем пентестеру знать топологию сети?
    <details><summary>Ответ</summary>Для понимания границ сети, поиска критичных серверов, планирования атак типа MITM или lateral movement</details>
3. Как работает traceroute?
    <details><summary>Ответ</summary>Отправляет пакеты с увеличивающимся TTL. Когда TTL=0, роутер отправляет ICMP Time Exceeded, так определяется каждый хоп</details>
4. Что такое маршрутизация по умолчанию?
    <details><summary>Ответ</summary>Маршрут для пакетов, для которых не найдено более конкретного маршрута (обычно ведет к провайдеру/интернету)</details>

## Задачи для самостоятельного выполнения

1. **Анализ маршрутизации до цели**: Выполните traceroute до 5 различных публичных IP-адресов (например, 8.8.8.8, 1.1.1.1, серверы в разных странах). Сравните количество хопов, задержки на каждом хопе. Определите, какие провайдеры/сети участвуют в маршруте (используйте whois для IP-адресов хопов).

2. **Создание нескольких маршрутов**: Настройте на тестовой машине (или виртуалке) два сетевых интерфейса (можно использовать dummy-интерфейсы). Добавьте маршруты для разных подсетей через разные интерфейсы. Проверьте, какой маршрут выбирается для конкретного IP с помощью `ip route get <ip>`.

3. **Перенаправление пакетов (IP forwarding)**: Включите IP forwarding на Linux:
```bash
sudo sysctl net.ipv4.ip_forward=1
```
Объясните, зачем это нужно и как это используется в атаках типа Man-in-the-Middle (например, с arpspoof).

4. **Анализ маршрутизации в захваченном трафике**: Захватите трафик traceroute с помощью tcpdump или Wireshark. Проанализируйте пакеты: как меняется поле TTL? Как выглядит ответ "Time to live exceeded"? Какой протокол используется по умолчанию (UDP)?

5. **Работа с маршрутизацией в nmap**: Используйте nmap с различными типами пинга для обнаружения хостов:
```bash
nmap -sn -PE 192.168.1.0/24   # ICMP Echo
nmap -sn -PS80 192.168.1.0/24  # TCP SYN на 80
nmap -sn -PA80 192.168.1.0/24  # TCP ACK на 80
```
Сравните результаты. Какой метод эффективнее при наличии файрволов?

## Адаптация под macOS (M2, 8GB)

Для пользователей macOS (особенно на чипах M1/M2 и с 8GB RAM):

- **Установка инструментов**: Используйте `brew install` вместо `apt install`:
  ```bash
  brew install nmap
  brew install mtr
  brew install traceroute
  brew install tcptraceroute
  ```

- **Сетевые команды на macOS**: В macOS нет `ip route`. Используйте:
  - `netstat -rn` — показать таблицу маршрутизации
  - `route -n get <ip>` — показать маршрут до конкретного IP
  - `ifconfig` — просмотр интерфейсов (устаревший, но работает)
  - Для `ip` команд: `brew install iproute2mac`

- **Виртуализация**: Вместо VirtualBox (который может быть нестабилен на M2) рекомендуется использовать:
  - **UTM** — нативный для Apple Silicon, бесплатный
  - **Parallels** — платный, но быстрый на M-чипах
  
  На 8GB RAM запускайте VM с 3-4GB памяти.

- **Устаревшие команды**: Везде, где в уроке упоминаются `ifconfig`, `netstat`, `arp` — эти команды считаются устаревшими. Используйте современные аналоги:
  - `ifconfig` → `ip addr` / `ip link`
  - `netstat -tunap` → `ss -tunap`
  - `arp -n` → `ip neigh`
  - `route -n` → `ip route`

- **Ограничения 8GB RAM**: Не запускайте одновременно много тяжелых VM. Оптимально: 1 Kali (3GB) + 1 маршрутизатор (1GB) = 4GB + хост ~4GB.
