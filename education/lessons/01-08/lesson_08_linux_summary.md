# Занятие 8: Итоговое практическое занятие — Linux для пентестера

## Теория

Итоговое занятие объединяет все изученные темы. Практика на реальных задачах платформы HackTheBox (Starting Point) — лучший способ закрепить навыки.

> **Примечание для macOS (M2, 8GB):** Для запуска Kali Linux на Mac M2 используйте UTM или Parallels Desktop вместо VirtualBox. Выделяйте виртуалке не более 3-4GB RAM при 8GB на хосте. Установка пакетов: `sudo apt install` (Kali), `brew install` (macOS).

### Что нужно для выполнения

1. Аккаунт на HackTheBox (https://hackthebox.com)
2. VPN-подключение к лаборатории HTB (оверлейный VPN) — *на машинах с 8GB RAM запускайте ВМ Kali с 3-4GB памяти*
3. Kali Linux (или аналогичный дистрибутив)
4. Базовые навыки работы с терминалом (занятия 1-7)

### Алгоритм работы с машиной на HTB

1. Подключиться к VPN: `sudo openvpn htb.ovpn`
2. Получить IP целевой машины из задания
3. Разведка: `nmap`, `ping`, проверка доступности
4. Перечисление служб и версий
5. Поиск уязвимостей и эксплуатация
6. Получение флагов (user.txt, root.txt)

### Полезные команды для HTB

```bash
# Проверка VPN
ip addr show tun0
```
**Пример вывода `ip addr show tun0`:**
```text
3: tun0: <POINTOPOINT,MULTICAST,NOARP,UP,LOWER_UP> mtu 1500 qdisc pfifo_fast state UNKNOWN group default qlen 500
    link/none
    inet 10.10.14.5/23 scope global tun0
       valid_lft forever preferred_lft forever
```

```bash
# Базовый скан nmap
nmap -sC -sV -oN scan.txt <TARGET_IP>
```

```bash
# Быстрая проверка веб-сервиса
curl -I http://<TARGET_IP>
```
**Пример вывода `curl -I http://10.10.10.5`:**
```text
HTTP/1.1 200 OK
Server: Apache/2.4.29 (Ubuntu)
Content-Type: text/html; charset=UTF-8
```

```bash
# Проверка SMB
smbclient -L //<TARGET_IP>/

# Поиск эксплойтов
searchsploit <service> <version>
```

## Практическое занятие

### Задачи на базовые навыки Linux (решаются на локальной машине Kali)

#### Задача 1: Навигация и файловая система
Создайте следующую структуру директорий в `~/.htb/`:
```
~/.htb/
├── machines/
│   ├── machine1/
│   │   ├── scans/
│   │   ├── exploits/
│   │   └── loot/
│   └── machine2/
│       ├── scans/
│       ├── exploits/
│       └── loot/
└── tools/
```
Запишите текущую дату и время в файл `~/.htb/machines/machine1/started.txt`.

#### Задача 2: Права доступа
В директории `~/.htb/machines/machine1/loot/` создайте файл `credentials.txt` с правами `600`. Попытайтесь прочитать его от имени другого пользователя (или проверьте права через `ls -l`). Убедитесь, что посторонние не имеют доступа.

#### Задача 3: Работа с процессами
Запустите простой Python-сервер в директории `~/.htb/`:
```bash
cd ~/.htb/
python3 -m http.server 8000 &
```
Найдите его PID через `ps aux | grep http.server`. Проверьте, что порт 8000 слушается через `ss -tlnp | grep 8000`. Корректно завершите процесс.

#### Задача 4: Сеть и разведка
Проверьте свой IP-адрес на интерфейсе `tun0` (или основном сетевом интерфейсе, если VPN не подключен). Сохраните вывод `ip addr show` в файл `~/network_info.txt`. Проверьте связность с `1.1.1.1` (3 пакета ping).

#### Задача 5: Поиск файлов
Найдите в системе все файлы, принадлежащие пользователю `root`, которые имеют SUID-бит и доступны для выполнения. Сохраните список в `~/suid_enum.txt`. Используйте `find / -perm -4000 -user root 2>/dev/null`.

#### Задача 6: Работа с текстом и grep
Создайте файл `services.txt` с выводом `ps aux`. Найдите все процессы, запущенные от имени `root`, и сохраните их в `~/root_processes.txt`. Посчитайте количество таких процессов.

#### Задача 7: Архивы и бэкапы
Заархивируйте директорию `~/.htb/machines/` в файл `~/htb_machines_backup.tar.gz`. Проверьте содержимое архива без распаковки (`tar -tzf`). Распакуйте архив в `/tmp/test_extract/`.

#### Задача 8: Bash-скрипт — автоматизация разведки
Напишите скрипт `htb_recon.sh`, который:
1. Принимает IP-адрес цели как аргумент
2. Создает директорию `recon_<IP>/`
3. Выполняет `nmap -sC -sV -oN recon_<IP>/nmap.txt <IP>`
4. Выполняет `ping -c 4 <IP>` и сохраняет в `recon_<IP>/ping.txt`
5. Выводит "Разведка завершена" по окончании

#### Задача 9: Анализ логов
Найдите в `/var/log/auth.log` (или аналогичном логе авторизации) все попытки входа (строки с "session opened" и "session closed"). Посчитайте количество входов каждого пользователя. Используйте `grep`, `awk`, `sort`, `uniq`.

#### Задача 10: Решение машины на HackTheBox (Starting Point)
Зарегистрируйтесь на HackTheBox, подключитесь к лаборатории через VPN. Выберите любую машину из раздела **Starting Point** (например, "Meow", "Fawn", "Dancing").

Выполните базовую разведку:
1. `nmap -sC -sV <TARGET_IP>`
2. Определите открытые порты и службы
3. Попробуйте получить доступ, следуя указаниям HTB
4. Найдите флаг(и) `user.txt` и/или `root.txt`

Запишите шаги, которые вы выполнили, в файл `~/htb_machine_walkthrough.txt`.

## Адаптация под macOS (M2, 8GB)

- Для установки инструментов используйте Homebrew: `brew install <tool>`

- Для Python используйте `pip3 install` вместо `pip install`


## Задачи для самостоятельного выполнения

1. Напишите скрипт `htb_full_recon.sh`, который объединяет: пинг, nmap (быстрый и полный), проверку веб-сервисов (curl), проверку SMB (smbclient). Скрипт должен принимать IP и сохранять всё в структурированную директорию.

2. Создайте alias'ы в `~/.bashrc` для часто используемых команд пентестера: `alias htb='cd ~/.htb'`, `alias nmap-quick='nmap -T4 -F'`, `alias nmap-full='nmap -p- -T4'`. Примените изменения через `source ~/.bashrc`.

3. Настройте приветствие в терминале (PS1), которое показывает текущего пользователя, хост и текущую директорию в цвете. Добавьте настройку в `~/.bashrc`.

4. Напишите скрипт `file_carver.sh`, который ищет в указанной директории все файлы определенного типа (например, .jpg, .png, .pdf) и копирует их в отдельную папку `~/found_files/`. Используйте `find` с `-name`.

5. Изучите базовые концепции следующих инструментов, которые понадобятся далее: `nmap`, `gobuster` (или `dirb`), `nikto`, `sqlmap`, `metasploit`. Напишите краткий cheat-sheet (по 3-5 основных команд для каждого) в файл `~/tools_cheatsheet.md`.

## Частые ошибки

1. **Запуск nmap без sudo.** Многие виды сканирования требуют прав суперпользователя для корректной работы.
2. **Забытое `&` при запуске веб-сервера.** `python3 -m http.server` без `&` блокирует терминал.
3. **Путаница между `tar -czf` и `tar -xzf`.** Первое создает архив, второе распаковывает. Перепутав флаги, можно затереть данные.
4. **Игнорирование `2>/dev/null`.** При поиске файлов от корня (find /) без перенаправления ошибок вывод будет засорен сообщениями "Permission denied".
