# Занятие 7: Bash-скрипты для пентестера

## Теория

Bash-скрипты автоматизируют рутинные задачи: сканирование, проверку хостов, сбор информации.

> **Примечание для macOS (M2, 8GB):** Для запуска Kali Linux на Mac M2 используйте UTM или Parallels Desktop вместо VirtualBox. Выделяйте виртуалке 3-4GB RAM при 8GB на хосте. Установка пакетов: `sudo apt install` (Kali), `brew install` (macOS). Bash-скрипты работают одинаково в Linux и macOS.

### Структура скрипта

```bash
#!/bin/bash
# Комментарий
команда1
команда2
```
Первая строка (`shebang`) указывает, какой интерпретатор использовать.

### Переменные

```bash
name="value"          # присваивание (без пробелов!)
echo $name            # использование
echo "${name}_suffix" # использование с фигурными скобками
```

### Аргументы скрипта

- `$0` — имя скрипта
- `$1`, `$2`, ... — аргументы
- `$#` — количество аргументов
- `$@` — все аргументы
- `$?` — код возврата последней команды (0 = успех)

### Условия (if)

```bash
if [ условие ]; then
    команды
elif [ условие ]; then
    команды
else
    команды
fi
```
Операторы сравнения:
- `-eq`, `-ne`, `-gt`, `-lt` — числовые
- `==`, `!=` — строковые
- `-f` — файл существует
- `-d` — директория существует
- `-z` — строка пуста

### Циклы

```bash
# for
for i in 1 2 3; do
    echo $i
done

# for с seq
for i in $(seq 1 10); do
    echo $i
done

# while
while [ условие ]; do
    команды
done
```

### Полезные конструкции

```bash
command1 && command2  # command2 выполнится, если command1 успешна
command1 || command2  # command2 выполнится, если command1 НЕ успешна
```

## Практическое занятие

### Шаг 1: Простейший скрипт

```bash
# Создать файл
cat > hello.sh << 'EOF'
#!/bin/bash
echo "Привет, пентестер!"
echo "Текущая дата: $(date)"
echo "Текущий пользователь: $(whoami)"
EOF

# Сделать исполняемым
chmod +x hello.sh

# Запустить
./hello.sh
```
**Пример вывода `./hello.sh`:**
```text
Привет, пентестер!
Текущая дата: Thu Jan 10 10:00:00 UTC 2024
Текущий пользователь: user
```

### Шаг 2: Скрипт проверки хостов (ping sweep)

```bash
cat > ping_sweep.sh << 'EOF'
#!/bin/bash

if [ -z "$1" ]; then
    echo "Использование: $0 <сеть> (например: 192.168.1)"
    exit 1
fi

network="$1"
echo "Сканирование сети ${network}.0/24..."

for i in $(seq 1 254); do
    ip="${network}.${i}"
    if ping -c 1 -W 1 "$ip" > /dev/null 2>&1; then
        echo "UP: $ip"
    fi
done
EOF

chmod +x ping_sweep.sh
# Тест (на своей сети!):
# ./ping_sweep.sh 192.168.1
```
**Пример вывода:**
```text
Сканирование сети 192.168.1.0/24...
UP: 192.168.1.1
UP: 192.168.1.105
```

### Шаг 3: Скрипт проверки портов (port scanner)

```bash
cat > port_check.sh << 'EOF'
#!/bin/bash

target="$1"
ports="22 80 443 445 3389 8080"

if [ -z "$target" ]; then
    echo "Использование: $0 <host>"
    exit 1
fi

echo "Проверка портов на $target..."
for port in $ports; do
    if timeout 1 bash -c "echo > /dev/tcp/$target/$port" 2>/dev/null; then
        echo "Порт $port: ОТКРЫТ"
    else
        echo "Порт $port: закрыт"
    fi
done
EOF

chmod +x port_check.sh
# Тест:
# ./port_check.sh 127.0.0.1
```
**Пример вывода:**
```text
Проверка портов на 127.0.0.1...
Порт 22: ОТКРЫТ
Порт 80: закрыт
Порт 443: закрыт
```

### Шаг 4: Скрипт сбора информации о системе

```bash
cat > recon.sh << 'EOF'
#!/bin/bash

target="$1"
output="recon_${target}.txt"

if [ -z "$target" ]; then
    echo "Использование: $0 <host>"
    exit 1
fi

echo "=== Разведка: $target ===" > "$output"
echo "Время: $(date)" >> "$output"
echo "" >> "$output"

echo "[*] Пинг..." >> "$output"
ping -c 3 "$target" >> "$output" 2>&1

echo "" >> "$output"
echo "[*] Проверка открытых портов..." >> "$output"
for port in 22 80 443 445 3389; do
    if timeout 1 bash -c "echo > /dev/tcp/$target/$port" 2>/dev/null; then
        echo "  Порт $port: ОТКРЫТ" >> "$output"
    fi
done

echo "" >> "$output"
echo "[*] DNS запрос..." >> "$output"
dig +short "$target" >> "$output" 2>/dev/null

echo "[+] Результат сохранен в $output"
EOF

chmod +x recon.sh
```

### Шаг 5: Обработка ошибок и логирование

```bash
cat > safe_script.sh << 'EOF'
#!/bin/bash

set -e          # выход при любой ошибке
set -u          # выход при использовании неопределенной переменной

log() {
    echo "[$(date '+%H:%M:%S')] $1"
}

log "Скрипт запущен"

if [ ! -f "/etc/passwd" ]; then
    log "ОШИБКА: файл не найден"
    exit 1
fi

log "Работа выполнена успешно"
EOF

chmod +x safe_script.sh
./safe_script.sh
```
**Пример вывода `./safe_script.sh`:**
```text
[10:00:00] Скрипт запущен
[10:00:00] Работа выполнена успешно
```

## Адаптация под macOS (M2, 8GB)

- Для установки инструментов используйте Homebrew: `brew install <tool>`
- На MacBook Air M2 (8GB) запускайте VM с памятью не более 3-4GB
- Используйте UTM вместо VirtualBox (лучшая поддержка ARM)
- Docker работает нативно на M2: `docker pull <image>`
- Для VPN используйте Tunnelblick (OpenVPN) или официальные клиенты
- Для Python используйте `pip3 install` вместо `pip install`


## Задачи для самостоятельного выполнения

1. Напишите скрипт `file_backup.sh`, который принимает путь к файлу, создает его копию с добавлением даты в имя (например, `file_20260429.txt`), и сохраняет в `~/backups/`. Создайте директорию `~/backups/`, если её нет.

2. Напишите скрипт `user_enum.sh`, который читает `/etc/passwd` и выводит список пользователей, у которых: (а) интерактивная оболочка (bash/sh/zsh), (б) UID >= 1000 (обычные пользователи). Используйте `awk`.

3. Напишите скрипт `port_scanner.sh`, который принимает хост и список портов (через аргументы или переменную) и проверяет их доступность. Выводит только открытые порты. Используйте конструкцию `/dev/tcp`.

4. Напишите скрипт `log_analyzer.sh`, который принимает путь к лог-файлу и ищет в нем строки с кодами ошибок (слова "error", "failed", "denied" — без учета регистра). Выводит количество найденных строк для каждого типа.

5. Напишите скрипт `dir_enum.sh`, который принимает URL (например, `http://target.htb`) и список часто встречающихся путей (`/admin`, `/login`, `/dashboard`, `/config`) и проверяет их доступность через `curl -s -o /dev/null -w "%{http_code}"`. Выводит путь и код ответа.

## Частые ошибки

1. **Пробелы при присваивании переменных.** `name = "value"` вызовет ошибку. Пишите слитно: `name="value"`.
2. **Забытое `chmod +x`.** Скрипт не запустится с `./script.sh`, если не сделать его исполняемым.
3. **Отсутствие shebang `#!/bin/bash`.** Без неё скрипт может выполняться не той оболочкой, что приведет к ошибкам.
4. **Использование `$?` после другой команды.** `$?` содержит код возврата *последней выполненной* команды, сохраняйте его сразу после целевой команды.
