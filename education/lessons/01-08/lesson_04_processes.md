# Занятие 4: Процессы и службы в Linux

## Теория

Процесс — запущенная программа. В Linux каждый процесс имеет PID (Process ID), владельца и состояние.

> **Примечание для macOS (M2, 8GB):** При запуске Kali Linux на Mac M2 используйте UTM или Parallels Desktop. Выделяйте виртуалке 3-4GB RAM при 8GB на хосте. Установка пакетов: `sudo apt install` (Kali), `brew install` (macOS).

### Состояния процессов

- **Running (R)** — выполняется или готов к выполнению
- **Sleeping (S)** — ожидает события (спит)
- **Stopped (T)** — остановлен (сигнал SIGSTOP)
- **Zombie (Z)** — завершился, но не удален из таблицы процессов

### Иерархия процессов

- PID 1 — `systemd` (или `init`) — родитель всех процессов
- Дочерние процессы наследуются от родительских
- `PPID` — Parent Process ID

### Основные команды

| Команда | Назначение |
|---------|-----------|
| `ps` | Список процессов |
| `top` | Процессы в реальном времени (классика) |
| `htop` | Улучшенный top (нужна установка) |
| `systemctl` | Управление службами systemd |
| `kill` | Отправка сигналов процессам |
| `pkill` | Убить процесс по имени |
| `jobs` | Фоновые задачи текущей оболочки |
| `bg` / `fg` | Перевод задач в фон/на передний план |

### Сигналы

- `SIGTERM (15)` — запрос на завершение (по умолчанию)
- `SIGKILL (9)` — принудительное завершение (нельзя перехватить)
- `SIGSTOP (19)` — остановка процесса
- `SIGCONT (18)` — продолжение выполнения

### systemd

Современная система инициализации. Управляет службами (services), точками монтирования, сокетами.

```
systemctl status служба    # статус
systemctl start служба     # запуск
systemctl stop служба      # остановка
systemctl enable служба    # автозапуск
systemctl disable служба   # отключить автозапуск
```

## Практическое занятие

### Шаг 1: Просмотр процессов (ps)

```bash
# Процессы текущего пользователя
ps
```
**Пример вывода `ps`:**
```text
  PID TTY          TIME CMD
 1234 pts/0    00:00:00 bash
 5678 pts/0    00:00:00 ps
```

```bash
# Все процессы с полным форматом
ps aux
```
**Пример вывода `ps aux` (фрагмент):**
```text
USER       PID %CPU %MEM    VSZ   RSS TTY      STAT START   TIME COMMAND
root         1  0.0  0.1  22568  5120 ?        Ss   09:00   0:01 /sbin/init
root       123  0.0  0.2  72340  8192 ?        Ss   09:00   0:00 sshd: /usr/sbin/sshd
user      1234  0.0  0.1  18520  4096 pts/0    Ss   09:30   0:00 bash
```

```bash
# Дерево процессов
ps auxf

# Процессы конкретного пользователя
ps -u root

# Только PID, имя и команда
ps -eo pid,user,cmd
```

### Шаг 2: Мониторинг в реальном времени (top / htop)

```bash
# Запустить top
top
# Клавиши: q — выход, k — kill процесса, r — renice, 1 — показать все CPU

# Установить и запустить htop (если нет)
# Для Kali Linux:
sudo apt install -y htop
# Для macOS (M2):
brew install htop
htop
# Клавиши: F9 — kill, F6 — сортировка, q — выход
```

### Шаг 3: Управление процессами (kill)

```bash
# Запустить процесс в фоне
sleep 1000 &

# Найти его PID
ps aux | grep sleep
```
**Пример вывода `ps aux | grep sleep`:**
```text
user  5678  0.0  0.0   5912   896 pts/0    S    10:00   0:00 sleep 1000
```

```bash
# Отправить SIGTERM (мягкое завершение)
kill <PID>

# Если не завершается — SIGKILL
kill -9 <PID>

# Убить по имени
pkill sleep

# Убить всех пользователей с именем процесса
pkill -u username process_name
```

### Шаг 4: Фоновые задачи

```bash
# Запустить задачу в фоне
sleep 500 &

# Посмотреть фоновые задачи
jobs
```
**Пример вывода `jobs`:**
```text
[1]+  Running                 sleep 500 &
```

```bash
# Остановить задачу (Ctrl+Z) и перевести в фон
sleep 400
# Ctrl+Z (остановит)
bg  # продолжит в фоне

# Вернуть на передний план
fg

# Запустить и сразу отвязать от терминала
nohup sleep 1000 &
```

### Шаг 5: Управление службами (systemctl)

```bash
# Список всех служб
systemctl list-units --type=service

# Список запущенных служб
systemctl list-units --type=service --state=running

# Статус конкретной службы
systemctl status ssh
```
**Пример вывода `systemctl status ssh`:**
```text
● ssh.service - OpenBSD Secure Shell server
     Loaded: loaded (/lib/systemd/system/ssh.service; enabled; preset: enabled)
     Active: active (running) since Mon 2024-01-10 09:00:00 UTC; 1h ago
       Docs: man:sshd(8)
             man:sshd_config(5)
   Main PID: 123 (sshd)
      Tasks: 1 (limit: 4617)
     Memory: 5.0M
        CPU: 100ms
     CGroup: /system.slice/ssh.service
             └─123 "sshd: /usr/sbin/sshd -D [listener] 0 of 10-100 startups"
```

```bash
# Остановить службу
sudo systemctl stop ssh

# Запустить службу
sudo systemctl start ssh

# Отключить автозапуск
sudo systemctl disable ssh

# Включить автозапуск
sudo systemctl enable ssh

# Перезапустить
sudo systemctl restart ssh
```

## Адаптация под macOS (M2, 8GB)

- Для установки инструментов используйте Homebrew: `brew install <tool>`
- На MacBook Air M2 (8GB) запускайте VM с памятью не более 3-4GB
- Используйте UTM вместо VirtualBox (лучшая поддержка ARM)
- Docker работает нативно на M2: `docker pull <image>`

- Для Python используйте `pip3 install` вместо `pip install`


## Задачи для самостоятельного выполнения

1. Найдите процесс, который потребляет больше всего оперативной памяти, используя `ps aux --sort=-%mem | head`. Запишите его имя, PID и потребление памяти.

2. Запустите простой веб-сервер в фоне: `python3 -m http.server 8080 &`. Найдите его PID, проверьте статус через `ps`, затем корректно завершите процесс через `kill` (без -9).

3. Создайте скрипт `loop.sh`, который бесконечно выводит текущее время в файл `/tmp/time.log` раз в секунду. Запустите его в фоне, убедитесь что он работает, затем остановите через `pkill` по имени. *Примечание: создание bash-скриптов подробно изучается в Уроке 7. Кратко: создайте файл со строкой `while true; do date >> /tmp/time.log; sleep 1; done` и сделайте его исполняемым.*

4. Исследуйте службу SSH (`ssh` или `sshd`). Узнайте, запущена ли она, включен ли автозапуск. Если служба не запущена — запустите её и включите автозапуск.

5. Найдите все процессы, запущенные от имени пользователя `root`, которые слушают сетевые порты. Используйте комбинацию `ps aux` и `grep` или изучите утилиту `ss` в Уроке 5. Кратко: `ss -tulpn` покажет слушающие порты и процессы. Сохраните список в `~/root_network_processes.txt`.

## Частые ошибки

1. **Использование `kill -9` как первого средства.** SIGKILL не дает процессу корректно завершиться. Сначала используйте `kill` (SIGTERM), и только если не помогло — `kill -9`.
2. **Забытое `&` при запуске в фоне.** Без `&` процесс занимает терминал и блокирует ввод до завершения.
3. **Путаница между PID и PPID.** PID — это ID самого процесса, PPID — ID родительского процесса.
4. **Остановка служб без понимания.** Остановка `ssh` или `NetworkManager` может разорвать соединение с машиной, особенно в пентест-лаборатории.
