# Занятие 3: Права доступа в Linux

## Теория

В Linux каждый файл и директория имеют права доступа, определяющие кто может читать, записывать и выполнять их.

### Типы прав доступа

Права обозначаются тремя группами по три бита: `rwx` (read, write, execute)

```
-rwxr-xr--  1 user group  4096 file.txt
↑  ↑↑↑ ↑↑↑ ↑↑↑
│   │││ │││ │││
│   │││ │││ └└└─ права для "others" (остальные)
│   │││ ││└──── права для "group" (группа)
│   │││ └└───── права для "owner" (владелец)
│   └└└─────── тип файла и права
└───────────── тип файла (- обычный, d директория, l ссылка)
```

### Числовое представление (octal)

| Бит | Значение | Число |
|-----|----------|-------|
| r   | чтение   | 4     |
| w   | запись   | 2     |
| x   | выполнение | 1   |

Примеры:
- `755` = rwxr-xr-x (владелец всё, остальные только чтение и выполнение)
- `644` = rw-r--r-- (владелец читает/пишет, остальные только читают)
- `700` = rwx------ (только владелец имеет доступ)

### Ключевые команды

| Команда | Назначение |
|---------|-----------|
| `chmod` | Изменить права доступа |
| `chown` | Изменить владельца и группу |
| `sudo` | Выполнить команду от имени root |
| `su` | Переключиться на другого пользователя |
| `id` | Показать UID, GID, группы |
| `whoami` | Показать текущего пользователя |

### SUID, SGID, Sticky bit

- **SUID** (4000) — программа выполняется с правами владельца (например, `/usr/bin/passwd`)
- **SGID** (2000) — программа выполняется с правами группы
- **Sticky bit** (1000) — в директории (например `/tmp`) файлы может удалить только владелец

## Практическое занятие

### Шаг 1: Проверка текущего пользователя и прав

```bash
# Кто я?
whoami

# Мои права и группы
id

# Список пользователей в системе
cat /etc/passwd

# Список групп
cat /etc/group
```

### Шаг 2: Просмотр и изменение прав (chmod)

```bash
cd ~/terminal_practice

# Создать тестовый файл
echo "secret data" > secret.txt

# Посмотреть права
ls -l secret.txt

# Убрать все права для others
chmod o-rwx secret.txt
ls -l secret.txt

# Дать все права владельцу, никаких для остальных
chmod 700 secret.txt
ls -l secret.txt

# Сделать файл исполняемым
echo '#!/bin/bash' > script.sh
echo 'echo "Hello"' >> script.sh
chmod +x script.sh
ls -l script.sh
./script.sh
```

### Шаг 3: Изменение владельца (chown)

```bash
# Посмотреть текущего владельца
ls -l secret.txt

# Создать тестового пользователя (если нет)
sudo useradd testuser

# Изменить владельца файла
sudo chown testuser secret.txt
ls -l secret.txt

# Изменить владельца и группу
sudo chown testuser:testuser secret.txt
ls -l secret.txt

# Вернуть себе
sudo chown $USER:$USER secret.txt
```

### Шаг 4: Работа с sudo и su

```bash
# Выполнить команду от root
sudo cat /etc/shadow

# Переключиться на root (нужен пароль root)
sudo -i
# или
su -
# Выйти из root
exit

# Посмотреть, кто может использовать sudo
cat /etc/sudoers
sudo cat /etc/sudoers
```

### Шаг 5: SUID и поиск уязвимостей

```bash
# Найти все файлы с SUID-битом (потенциальные векторы повышения привилегий)
find / -perm -4000 2>/dev/null

# Посмотреть SUID у конкретного файла
ls -l /usr/bin/passwd

# Найти файлы с правами записи для всех
find / -perm -002 2>/dev/null | head -20
```

## Задачи для самостоятельного выполнения

1. Создайте файл `private.txt` и установите права `600` (только владелец может читать и писать). Проверьте, что другой пользователь (или от имени другого пользователя через `su`) не может прочитать файл.

2. Создайте директорию `shared/` и установите права `775`. Создайте внутри файл `team.txt` с правами `664`. Объясните, почему выбраны такие права.

3. Найдите в системе все файлы с установленным SUID-битом. Сохраните список в `~/suid_files.txt`. Исследуйте, какие из них могут быть использованы для повышения привилегий (nmap, find, vim с SUID — критично).

4. Создайте скрипт `backup.sh`, который архивирует директорию `~/practice` в `~/backup.tar.gz`. Сделайте скрипт исполняемым только для владельца. Запустите скрипт через `sudo` и убедитесь, что архив создан.

5. Исследуйте файл `/etc/sudoers` и файлы в `/etc/sudoers.d/`. Узнайте, как дать пользователю право запускать конкретную команду без ввода пароля. Напишите пример строки для sudoers.
