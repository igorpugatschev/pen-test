# Занятие 1: Введение в Linux для пентестера

## Теория

Linux — основная ОС для пентестера. Дистрибутив Kali Linux включает готовый набор инструментов для тестирования на проникновение.

### Основные понятия

**Дистрибутивы Linux:**
- Kali Linux — специализированный дистрибутив для пентеста (основан на Debian)
- Parrot Security — альтернатива Kali с акцентом на приватность
- Ubuntu/Debian — для общего использования и серверов

**Файловая система Linux:**
- Все начинается с корня `/`
- Нет дисков C:, D: — всё в едином дереве
- Регистр имеет значение: `File.txt` и `file.txt` — разные файлы

**Ключевые директории:**
- `/etc` — конфигурационные файлы системы и служб
- `/var` — изменяемые данные (логи, кэш, почта, базы данных)
- `/home` — домашние директории пользователей
- `/root` — домашняя директория суперпользователя
- `/bin`, `/usr/bin` — исполняемые файлы программ
- `/tmp` — временные файлы (часто слабые права доступа)
- `/proc`, `/sys` — виртуальные файловые системы (информация о процессах и ядре)

**Виртуализация:**
- VMware Workstation / VMware Fusion
- VirtualBox (бесплатный)
- Прямой запуск на железе или в облаке

## Практическое занятие

### Шаг 1: Установка Kali Linux в VM

1. Скачать образ Kali Linux (VirtualBox/VMware) с официального сайта kali.org
2. Создать новую виртуальную машину:
   - Тип: Linux, версия: Debian (64-bit)
   - ОЗУ: минимум 2 ГБ (рекомендуется 4 ГБ)
   - Диск: 20-40 ГБ (динамический)
3. Загрузить VM с ISO-образа
4. Выбрать графическую установку (Graphical Install)
5. Настроить сеть, пароль root, создать пользователя
6. Завершить установку и перезагрузиться

### Шаг 2: Первичная настройка

```bash
# Обновление системы
sudo apt update && sudo apt full-upgrade -y

# Установка базовых инструментов
sudo apt install -y vim curl wget git

# Проверка версии Kali
cat /etc/os-release
```

### Шаг 3: Изучение файловой системы

```bash
# Перейти в корень и посмотреть структуру
cd /
ls -la

# Изучить содержимое ключевых директорий
ls /etc/
ls /var/
ls /home/

# Посмотреть логи системы
ls /var/log/

# Найти конфиг сети
ls /etc/network/
cat /etc/hostname
cat /etc/hosts
```

### Шаг 4: Навигация по файловой системе

```bash
# Где я сейчас?
pwd

# Перейти в домашнюю директорию
cd ~
pwd

# Перейти в /etc
cd /etc
pwd

# Вернуться назад
cd ..
pwd

# Перейти в домашнюю директорию пользователя root
cd /root
# (если нет прав, используйте sudo -i)
```

## Задачи для самостоятельного выполнения

1. Установите Kali Linux в виртуальную машину и создайте снимок (snapshot) чистой системы.

2. Изучите содержимое директории `/etc`. Найдите файлы: `passwd`, `shadow`, `group`, `hosts`, `resolv.conf`. Опишите назначение каждого файла в текстовом файле `etc_files.txt`.

3. Создайте структуру директорий в своем домашнем каталоге:
   ```
   ~/pen-test/
   ├── targets/
   ├── tools/
   ├── reports/
   └── scripts/
   ```
   Используйте команду `mkdir -p` для создания вложенных директорий.

4. Найдите в `/var/log/` файлы, измененные за последние 24 часа. Используйте команду `find` с параметром `-mtime`. Запишите результаты в файл `recent_logs.txt`.

5. Выведите полный путь к исполняемому файлу команды `python3` с помощью команды `which` и `whereis`. Сравните результаты.
