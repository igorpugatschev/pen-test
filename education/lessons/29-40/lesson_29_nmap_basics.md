# Урок 29: Nmap основы

## Теория

Nmap (Network Mapper) — самый популярный инструмент для сканирования сети и аудита безопасности. Позволяет обнаруживать хосты, открытые порты, версии сервисов и операционные системы.

Основные типы сканирования:
- **TCP Connect Scan (-sT)** — полное TCP-соединение
- **SYN Stealth Scan (-sS)** — полустандартное сканирование (требует root)
- **UDP Scan (-sU)** — сканирование UDP-портов
- **Version Detection (-sV)** — определение версий сервисов
- **OS Detection (-O)** — определение ОС

## Практическое занятие

### Установка
```bash
# Kali Linux (уже установлен)
sudo apt update && sudo apt install nmap

# macOS (M2, Homebrew)
brew install nmap

# Проверка версии
nmap --version
# Пример вывода:
# Nmap version 7.94 ( https://nmap.org )
# Platform: x86_64-apple-darwin22.6.0
```

### Базовое сканирование
```bash
# Сканирование одного хоста
nmap 192.168.1.1
# Пример вывода:
# Starting Nmap 7.94 ( https://nmap.org )
# Nmap scan report for 192.168.1.1
# Host is up (0.001s latency).
# Not shown: 998 filtered tcp ports
# PORT   STATE SERVICE
# 80/tcp open  http

# Сканирование диапазона IP
nmap 192.168.1.1-100

# Сканирование подсети
nmap 192.168.1.0/24
```

### Основные флаги
```bash
# SYN сканирование (быстрое, stealth)
sudo nmap -sS 192.168.1.1
# Пример вывода:
# Starting Nmap 7.94
# Nmap scan report for 192.168.1.1
# PORT   STATE SERVICE
# 22/tcp open  ssh
# 80/tcp open  http

# Определение версий сервисов
nmap -sV 192.168.1.1
# Пример вывода:
# PORT   STATE SERVICE VERSION
# 22/tcp open  ssh     OpenSSH 8.2p1 Ubuntu
# 80/tcp open  http    Apache httpd 2.4.41

# Определение ОС
sudo nmap -O 192.168.1.1
# Пример вывода:
# OS details: Linux 5.4.0-90-generic

# Агрессивное сканирование (все основные проверки)
nmap -A 192.168.1.1

# Сканирование конкретных портов
nmap -p 22,80,443 192.168.1.1
nmap -p 1-1000 192.168.1.1
nmap -p- 192.168.1.1  # Все порты (1-65535)
```

### Сохранение результатов
```bash
# Обычный вывод
nmap -oN scan.txt 192.168.1.1

# XML формат (для автоматизации)
nmap -oX scan.xml 192.168.1.1
# Пример вывода (в файле):
# <?xml version="1.0"?>
# <nmaprun scanner="nmap" args="nmap -oX scan.xml 192.168.1.1" ...>

# Все форматы сразу
nmap -oA scan_all 192.168.1.1
```

## Адаптация под macOS (M2, 8GB)

- Для установки инструментов используйте Homebrew: `brew install <tool>`
- На MacBook Air M2 (8GB) запускайте VM с памятью не более 3-4GB
- Используйте UTM вместо VirtualBox (лучшая поддержка ARM)
- Docker работает нативно на M2: `docker pull <image>`
- Для VPN используйте Tunnelblick (OpenVPN) или официальные клиенты
- Для Python используйте `pip3 install` вместо `pip install`

## Задачи для самостоятельного выполнения

1. Установите Metasploitable2 в VM. Выполните базовое сканирование (nmap -sS) и найдите все открытые TCP порты.

2. Используя флаг -sV, определите версии сервисов на портах 21, 22, 23, 80, 445. Напишите, какие сервисы и версии запущены.

3. Выполните агрессивное сканирование (-A) целевого хоста. Сохраните результат в три формата (-oA). Проанализируйте вывод — что удалось обнаружить?

4. Напишите скрипт на Bash, который принимает IP-адрес как аргумент и выполняет:
   - Быстрое сканирование топ-100 портов
   - Определение версий сервисов
   - Сохранение в файл results.txt

5. Изучите вывод `nmap --help`. Найдите флаг для сканирования с обходом Firewall/IDS (-f, -D, -g). Попробуйте применить один из них.

## Частые ошибки

1. **Запуск SYN-сканирования без прав root** — флаг `-sS` требует прав суперпользователя, используйте `sudo`.

2. **Сканирование всех портов (-p-) без необходимости** — это долго (может занять часы), используйте для быстрой проверки `-p 1-1000`.

3. **Забыт флаг `-sV` для определения версий** — без него вы видите только номера портов, но не знаете, какая версия сервиса запущена.

4. **Игнорирование файрволов** — некоторые хосты могут блокировать сканирование, используйте `-Pn` для пропуска проверки на доступность.
