# Занятие 61. PTES стандарт: этапы Pre-engagement → Reporting

## Теория

PTES (Penetration Testing Execution Standard) — отраслевой стандарт проведения пентестов, определяющий методологию от начала до конца.

### Семь этапов PTES

**1. Pre-engagement Interactions (Предварительные договоренности)**
- Определение scope (области тестирования)
- Согласование правил (Rules of Engagement)
- Юридические аспекты: NDA, контракт
- Определение целей тестирования
- Классификация теста: black box, white box, gray box
- Временные рамки и окна обслуживания

**2. Intelligence Gathering (Сбор разведданных)**
- OSINT: публичные источники, соцсети, DNS
- Активный сбор: сканирование, enumeration
- Пассивный сбор: архивы, сертификаты, репозитории
- Картирование атакующей поверхности

**3. Threat Modeling (Моделирование угроз)**
- Идентификация активов (assets)
- Определение векторов атак
- Оценка вероятности и влияния
- Приоритизация целей

**4. Vulnerability Analysis (Анализ уязвимостей)**
- Автоматизированное сканирование
- Ручная проверка находок
- Валидация ложноположительных срабатываний
- Категоризация по CVSS

**5. Exploitation (Эксплуатация)**
- Подбор и адаптация эксплойтов
- Обход защитных механизмов (AV, EDR, WAF)
- Доказательство концепции (PoC)
- Оценка реального риска

**6. Post-Exploitation (Пост-эксплуатация)**
- Повышение привилегий
- Lateral movement
- Сбор конфиденциальных данных
- Persistence (закрепление)
- Оценка бизнес-рисков

**7. Reporting (Отчетность)**
- Executive Summary для руководства
- Technical Report для технических специалистов
- Рекомендации по устранению
- Приложения с доказательствами

### Ключевые документы PTES
- Rules of Engagement (RoE)
- Penetration Testing Agreement
- Final Report

## Практическое занятие

### Создание шаблона Rules of Engagement

Создайте документ `roe_template.md` со следующей структурой:

```markdown
# Rules of Engagement (RoE)
## Документ согласования условий пентеста

### 1. Общая информация
- Заказчик: [Название компании]
- Исполнитель: [Название компании пентестеров]
- Даты проведения: с [дата] по [дата]
- Контактные лица: [ФИО, должность, телефон, email]

### 2. Область тестирования (Scope)
#### Включено (In-Scope):
- IP-адреса: [список]
- Домены: [список]
- Приложения: [список]
- Физические объекты: [если применимо]

#### Исключено (Out-of-Scope):
- [Список систем, тестирование которых запрещено]
- Критические системы в часы работы
- Атака на инфраструктуру третьих лиц

### 3. Тип тестирования
- [ ] Black Box (без знания внутреннего устройства)
- [ ] Gray Box (частичное знание)
- [ ] White Box (полное знание, предоставление документации)

### 4. Разрешенные методы
- [ ] Сканирование портов и сервисов
- [ ] Тестирование на уязвимости (vuln scanning)
- [ ] Эксплуатация уязвимостей
- [ ] Социальная инженерия (фишинг, звонки)
- [ ] Физический доступ
- [ ] DoS/DDoS атаки (НЕТ)
- [ ] Brute-force паролей (ограниченно)

### 5. Ограничения
- Запрет на доступ к системам, не входящим в scope
- Уведомление при обнаружении критических уязвимостей
- Ограничение нагрузки на системы (не более X параллельных запросов)
- Запрет на модификацию критических данных

### 6. Эскалация инцидентов
При обнаружении критических уязвимостей (CVSS 9.0+):
1. Немедленный звонок: [телефон]
2. Email: [email]
3. Остановка тестирования до получения указаний

### 7. Конфиденциальность
- Все данные являются конфиденциальными
- NDA подписан: [дата]
- Уничтожение данных после отчета: [срок]

### 8. Разрешение на тестирование
Подтверждаю право на проведение пентеста в указанных рамках:

_________________ / [ФИО] / Дата: _________
(Подпись заказчика)
```

### Практика: заполнение RoE
Заполните шаблон для гипотетической компании "Ромашка Банк":
- Scope: 192.168.1.0/24, домен romashka-bank.ru
- Тип: Gray Box
- Разрешено: сканирование, эксплуатация, социальная инженерия (только фишинг)
- Запрещено: DoS, физический доступ

## Задачи для самостоятельного выполнения

1. **Сравнение методологий**: Составьте таблицу сравнения PTES, OWASP Testing Guide и NIST SP 800-115. Укажите сильные и слабые стороны каждой.

2. **Scope definition**: Для интернет-магазина "TechShop" определите scope пентеста. Магазин имеет: веб-сайт, мобильное приложение, API, внутреннюю сеть офиса, облачную инфраструктуру в AWS. Что включите в scope, а что нет? Обоснуйте.

3. **Threat Modeling**: Проведите упрощенное моделирование угроз для типичного веб-приложения интернет-банкинга. Выделите топ-5 активов и топ-5 векторов атак.

4. **RoE Checklist**: Создайте чек-лист проверки RoE перед началом пентеста (минимум 15 пунктов).

5. **Timeline planning**: Спланируйте таймлайн пентеста для средней компании (500 сотрудников, 10 веб-приложений, внутренняя сеть). Распределите время по этапам PTES.
