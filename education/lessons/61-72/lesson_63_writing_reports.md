# Занятие 63. Написание отчетов: шаблоны Executive Summary, Technical Details

## Теория

Отчет о пентесте — главный продукт работы. От качества отчета зависит, будут ли устранены уязвимости и пригласят ли вас снова.

### Структура отчета о пентесте

1. **Executive Summary** (Резюме для руководства)
2. **Introduction** (Введение)
3. **Methodology** (Методология)
4. **Findings / Vulnerabilities** (Технические детали)
5. **Risk Assessment** (Оценка рисков)
6. **Recommendations** (Рекомендации)
7. **Appendices** (Приложения)

### Принципы хорошего отчета
- **Executive Summary** пишется для бизнеса: на русском/английском языке, без технического сленга, акцент на бизнес-рисках
- **Technical Details** пишутся для технических специалистов: точные шаги воспроизведения, скриншоты, код
- **Recommendations** должны быть конкретными и выполнимыми
- Используйте CVSS для оценки серьезности
- Прикладывайте доказательства (screenshots, logs, PoC)

### Типы аудитории
| Аудитория | Что важно | Уровень детализации |
|-----------|-----------|---------------------|
| CEO/CISO | Бизнес-риск, деньги, репутация | Высокий уровень, без техдеталей |
| IT Director | Влияние на инфраструктуру | Средний уровень |
| SysAdmin/SecOps | Как исправить | Низкий уровень, максимум техдеталей |

## Ключевые риски для бизнеса

### 1. Несанкционированный доступ к базе данных клиентов
**Риск:** Утечка персональных данных 500 000+ клиентов  
**Влияние:** Штрафы до 18 млн руб. (152-ФЗ), репутационный ущерб, потеря доверия  
**Причина:** SQL-инъекция в форме поиска на сайте  
**Рекомендация:** Использование подготовленных выражений (prepared statements)  

### 2. Компрометация внутренней сети
**Риск:** Полный контроль над доменом (Domain Admin)  
**Влияние:** Остановка бизнес-процессов, кража данных, вымогатели  
**Причина:** Использование одинаковых паролей на серверах  
**Рекомендация:** Внедрение LAPS, уникальные пароли локальных администраторов  

### 3. Недостаточная защита веб-приложения
**Риск:** Дефейс сайта, кража сессий клиентов  
**Влияние:** Репутационный ущерб, потеря клиентов  
**Причина:** Отсутствие заголовков безопасности, XSS-уязвимости  
**Рекомендация:** Внедрение CSP, HSTS, валидация ввода

### Шаблон Technical Details (Vulnerability Entry)

```markdown
# Technical Report: Детали уязвимостей

## Уязвимость #1: SQL Injection в форме поиска

### Описание
В форме поиска на странице https://romashka-bank.ru/search.php обнаружена уязвимость SQL-инъекции. Параметр `query` не проходит должной валидации, что позволяет выполнять произвольные SQL-запросы к базе данных.

### CVSS Score
**CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:N**  
**Base Score: 9.1 (Critical)**

### Доказательство концепции (PoC)

#### Запрос:
```http
POST /search.php HTTP/1.1
Host: romashka-bank.ru
Content-Type: application/x-www-form-urlencoded

query=' UNION SELECT database(), user(), version()--
```

#### Ответ сервера:
```html
Результаты поиска:
База данных: romashka_db
Пользователь: db_user@localhost
Версия: MySQL 5.7.36
```

#### Извлечение данных пользователей:
```sql
' UNION SELECT username, password, email FROM users--
```

Результат:
| username | password (hash) | email |
|----------|-----------------|-------|
| admin | $2y$10$abcd... | admin@romashka-bank.ru |
| user1 | $2y$10$efgh... | user1@mail.ru |
| ... | ... | ... |

### Влияние
- Полный доступ к базе данных
- Утечка хешей паролей (возможен brute-force)
- Доступ к персональным данным клиентов
- Возможна модификация или удаление данных

### Рекомендации по устранению
1. Использовать подготовленные выражения (prepared statements):
```php
// Неправильно:
$sql = "SELECT * FROM products WHERE name = '$query'";

// Правильно:
$stmt = $pdo->prepare("SELECT * FROM products WHERE name = ?");
$stmt->execute([$query]);
```

2. Использовать ORM (Doctrine, Eloquent)
3. Включить WAF (ModSecurity) как временную меру
4. Минимизировать привилегии пользователя БД (только SELECT для поиска)

### Статус
- [ ] Не устранено
- [ ] Устранено (требует проверки)
- [x] Частично устранено

### Приложения
- Скриншот_1.png: Форма поиска
- Скриншот_2.png: Результат инъекции
- sqlmap_output.txt: Вывод автоматизированного сканирования
```
