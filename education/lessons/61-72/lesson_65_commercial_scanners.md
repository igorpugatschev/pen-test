# Занятие 65. Коммерческие сканеры: Nessus Home, изучение интерфейса

## Теория

Коммерческие сканеры уязвимостей — это автоматизированные инструменты, которые помогают быстро выявить известные уязвимости в инфраструктуре.

### Обзор популярных коммерческих сканеров

| Сканер | Вендор | Тип | Цена | Особенности |
|--------|--------|-----|------|-------------|
| **Nessus** | Tenable | Сетевой сканер | Бесплатно (Home), $3000+/год (Pro) | Надежный, большая база плагинов |
| **Qualys VMDR** | Qualys | Облачный | От $1500/год | SaaS, не требует установки |
| **Rapid7 InsightVM** | Rapid7 | Сетевой + облако | От $2000/год | Хорошая аналитика, Metasploit интеграция |
| **OpenVAS** | Greenbone | Open Source | Бесплатно | Альтернатива Nessus |
| **Nexpose** | Rapid7 | Сетевой | От $2000/год | Предшественник InsightVM |

### Nessus: возможности

**Nessus Professional (или Home для обучения):**
- Сканирование на уязвимости (vulnerability scanning)
- Проверка конфигурации (compliance checks)
- Локальное сканирование (credentialed scans)
- Аудит мобильных устройств
- Сканирование веб-приложений
- Интеграция с Nmap, Metasploit

**Типы сканирований в Nessus:**
1. **Basic Network Scan** — базовое сканирование сети
2. **Advanced Scan** — настраиваемое сканирование
3. **Credentialed Patch Audit** — проверка патчей (с учетными данными)
4. **Web Application Tests** — сканирование веб-приложений
5. **Malware Scan** — поиск вредоносного ПО
6. **Compliance Checks** — проверка соответствия стандартам (PCI DSS, CIS)

### Установка Nessus Home

Nessus Home — бесплатная версия для личного использования (до 16 IP-адресов).

**Шаги установки:**
1. Регистрация на https://www.tenable.com/products/nessus-home
2. Получение Activation Code
3. Скачивание для вашей ОС (Windows, macOS, Linux)
4. Установка и активация

**Для Kali Linux:**
```bash
# Скачать пакет
wget https://www.tenable.com/downloads/api/v2/pages/nessus/downloads/13394/download
dpkg -i Nessus-10.*.deb
/opt/nessus/sbin/nessuscli adduser
/opt/nessus/sbin/nessuscli fetch --register <activation-code>
systemctl start nessusd
```

**Доступ к интерфейсу:**
https://localhost:8834

### Интерфейс Nessus

**Главные разделы:**
1. **Scans** — управление сканированиями
2. **Policies** — настройка политик сканирования
3. **Plugins** — управление плагинами (база уязвимостей)
4. **Schedules** — расписание сканирований
5. **Settings** — настройки сканера

**Жизненный цикл сканирования:**
1. Создание нового сканирования (New Scan)
2. Выбор типа сканирования
3. Настройка целей (Targets)
4. Настройка учетных данных (Credentials) — опционально
5. Настройка расписания (Schedule) — опционально
6. Запуск (Launch)
7. Анализ результатов (Results)
8. Экспорт отчета (Export)

### Понимание результатов Nessus

**Уровни серьезности в Nessus:**
- **Critical** (Критический) — немедленное устранение
- **High** (Высокий) — устранить в ближайшее время
- **Medium** (Средний) — запланировать исправление
- **Low** (Низкий) — информирование
- **Info** (Информационный) — не требует действий

**Структура отчета об уязвимости:**
- Plugin ID — уникальный номер проверки
- Severity — серьезность
- CVSS Score — оценка CVSS
- Description — описание
- Solution — решение
- See Also — ссылки на источники
- Plugin Output — вывод сканера (доказательства)

## Практическое занятие

### Установка и настройка Nessus Home

Если у вас нет возможности установить Nessus, используйте альтернативу — OpenVAS (Greenbone Vulnerability Management).

**Установка OpenVAS в Kali Linux:**
```bash
sudo apt update
sudo apt install openvas
sudo gvm-setup
sudo gvm-check-setup
sudo runuser -u _gvm -- gvmd --create-user=admin --password=admin
sudo gvm-start
```

**Доступ к Greenbone Security Assistant:**
https://127.0.0.1:9392

### Практика: сканирование локальной сети

1. Запустите Nessus/OpenVAS
2. Создайте новое сканирование "Basic Network Scan"
3. Цель: ваша локальная сеть (например, 192.168.1.0/24) или отдельная машина (Metasploitable2, VulnHub VM)
4. Запустите сканирование
5. Дождитесь завершения (может занять 30-60 минут)

## Задачи для самостоятельного выполнения

1. **Сравнение сканеров**: Установите OpenVAS и выполните сканирование той же цели, что и Nessus (или наоборот). Сравните результаты: количество находок, ложноположительные срабатывания, удобство интерфейса.

2. **Credentialed Scan**: Настройте сканирование с учетными данными (Windows/Linux). Сравните результаты credentialed и non-credentialed сканирования. Что нового обнаружилось?

3. **Policy Tuning**: Создайте кастомную политику сканирования, отключив "шумные" проверки (например, DoS проверки). Запустите сканирование с этой политикой.

4. **Web Application Scan**: Настройте сканирование веб-приложения (например, DVWA или Juice Shop) через Nessus/OpenVAS. Сравните результаты с ручным тестированием по OWASP Top 10.

5. **False Positive Analysis**: Проанализируйте 5 уязвимостей из отчета сканера. Проверьте вручную (через nmap, nc, curl, браузер), являются ли они реальными. Напишите отчет с обоснованием.
