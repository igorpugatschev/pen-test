# Занятие 41: Python sockets — написать простой TCP-сканер портов

## Теория

Библиотека `socket` — это стандартная библиотека Python для работы с сетевыми соединениями на низком уровне. Она позволяет создавать клиентские и серверные сокеты, устанавливать TCP/UDP соединения, отправлять и получать данные.

**Зачем нужно:**
- Сканирование портов — основа разведки (reconnaissance) в пентесте
- Понимание того, как работают сетевые протоколы на уровне кода
- Создание собственных сетевых инструментов без внешних зависимостей

**Основные концепции:**
- `socket.AF_INET` — адресация IPv4
- `socket.SOCK_STREAM` — TCP сокет
- `socket.SOCK_DGRAM` — UDP сокет
- `connect_ex()` — попытка подключения, возвращает код ошибки (0 = успех)
- `settimeout()` — установка таймаута для операций

## Практическое занятие

Напишем простой TCP-сканер портов, который проверяет указанные порты на целевом хосте.

```python
import socket
import sys
from datetime import datetime

def tcp_port_scanner(target, ports):
    """
    Простой TCP-сканер портов
    :param target: целевой хост (IP или домен)
    :param ports: список портов для проверки
    """
    print(f"\n[+] Начало сканирования: {target}")
    print(f"[+] Время начала: {datetime.now()}\n")
    
    try:
        # Получаем IP-адрес цели
        target_ip = socket.gethostbyname(target)
        print(f"[+] IP адрес цели: {target_ip}\n")
    except socket.gaierror:
        print("[-] Ошибка разрешения имени хоста")
        return
    
    found_ports = []
    
    for port in ports:
        try:
            # Создаем TCP-сокет
            sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
            sock.settimeout(1)  # Таймаут 1 секунда
            
            # Пытаемся подключиться
            result = sock.connect_ex((target_ip, port))
            
            if result == 0:
                print(f"[+] Порт {port}: ОТКРЫТ")
                found_ports.append(port)
            else:
                print(f"[-] Порт {port}: закрыт")
            
            sock.close()
            
        except KeyboardInterrupt:
            print("\n[!] Сканирование прервано пользователем")
            sys.exit(0)
        except socket.error:
            print(f"[-] Ошибка соединения с портом {port}")
    
    print(f"\n[+] Сканирование завершено")
    print(f"[+] Найдено открытых портов: {len(found_ports)}")
    print(f"[+] Список открытых портов: {found_ports}")

if __name__ == "__main__":
    # Пример использования
    target_host = "127.0.0.1"  # Можно заменить на реальный хост
    ports_to_scan = [21, 22, 23, 25, 53, 80, 110, 143, 443, 445, 3389, 8080]
    
    tcp_port_scanner(target_host, ports_to_scan)
```

**Объяснение кода:**
1. Импортируем необходимые модули: `socket` для сетевых операций, `sys` для выхода, `datetime` для времени
2. Функция `tcp_port_scanner` принимает цель и список портов
3. `gethostbyname()` преобразует домен в IP-адрес
4. Для каждого порта создаем сокет, устанавливаем таймаут и пытаемся подключиться
5. `connect_ex()` возвращает 0, если соединение успешно (порт открыт)
6. Результаты выводятся на экран

**Запуск:**
```bash
python lesson_41_scanner.py
```

## Частые ошибки

1. **`socket.gaierror: [Errno 8] nodename nor servname provided`** — Означает, что доменное имя не может быть разрешено. Проверьте правильность написания цели.
2. **`TypeError: connect_ex() argument 1 must be a tuple (not int)`** — Метод `connect_ex` ожидает кортеж `(host, port)`, а передается два аргумента по отдельности. Правильно: `sock.connect_ex((target_ip, port))`.
3. **Игнорирование таймаута** — Без `sock.settimeout(1)` скрипт может зависнуть на неотвечающих портах на долгое время.
4. **Утечка сокетов** — Всегда используйте `sock.close()` после проверки порта. Лучше использовать контекстный менеджер `with`.

## Вопросы на понимание

1. **Чем отличается `connect()` от `connect_ex()`?**
   <details>
   <summary>Ответ</summary>
   `connect()` выбрасывает исключение `socket.error` при неудаче, а `connect_ex()` возвращает код ошибки (0 при успехе). `connect_ex()` удобнее для сканирования портов, так как не требует обработки исключений для закрытых портов.
   </details>

2. **Почему для сканирования портов лучше использовать TCP (`SOCK_STREAM`), а не UDP (`SOCK_DGRAM`)?**
   <details>
   <summary>Ответ</summary>
   UDP — это протокол без установления соединения. Сканеру сложнее определить состояние UDP порта (открыт/закрыт/фильтруется), так как часто ответа нет. TCP требует "рукопожатия", что делает определение состояния порта более надежным.
   </details>

3. **Как работает `socket.gethostbyname()` и почему важно вызывать его один раз перед циклом?**
   <details>
   <summary>Ответ</summary>
   Эта функция преобразует доменное имя в IP-адрес через DNS-запрос. Вызывать её в цикле для каждого порта неэффективно, так как это создаст лишнюю нагрузку на DNS-сервер и замедлит работу.
   </details>

## Адаптация под macOS (M2, 8GB)

При выполнении заданий на компьютере Mac с процессором M2 и 8GB оперативной памяти учитывайте следующие особенности:

- Для установки библиотек используйте `pip3 install <package>`, а не `pip install`.
- На M2 можно использовать `asyncio`, он поддерживает ARM.
- На 8GB RAM проект может быть тяжелым, используйте легковесные библиотеки.
- Для работы с aiohttp в будущих уроках используйте `connector = aiohttp.TCPConnector(ssl=False)` вместо передачи `ssl=False` напрямую в `get()`.
